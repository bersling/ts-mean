WRITE TYPESCRIPT EVERYWHERE

This project strives to provide a state-of-the art starter kit for your next web-app.

It's using the MEAN stack, which stands for

  • MongoDB
  • ExpressJS
  • Angular2+
  • NodeJS.

I also added a MySQL database layer, for those of you who prefer SQL to NoSQL for web-apps .

Now here comes the cool thing about this project: It's completely written in typescript.

Another requirement of this project is "100% test coverage" (also written in typescript!) for the starter kit. The project is available on Github!

Or maybe you are not conviced yet using the tsmean starter kit is a good idea? Then read on.

Why Typescript, why MEAN, why tsmean?

The MEAN stack was quite popular for a while, but it lost a bit of interest since the original projects aren't maintained anymore. Yet, I still think it's an incredible stack, now even more than ever. I see the main advantage in that there is just one language back to front. This brings efficiency gains through consistency:

  • All team members can develop fullstack
  • No context switching
  • Same packaging solutions, for example npm
  • Sharing code between backend and frontend (models, functions, ...)

The only problem with Javascript historically was, that it sucked. So why would you sacrifice good old Java, just to write crappy Javascript everywhere? Yet, with the advancements of Javascript and for me specifically with Typescript, one language everywhere became really appealing.

Additionally, compared to other fullstackJS solutions (e.g. MeteorJS), I think this stack comes with a more modular design. Don't like MongoDB? Just switch it out with another database.

Let's investigate each part of "MEAN" a bit more in detail. So the "M" I've already mentioned I actually lean more towards the MqSQL solution that the Mongo solution. With the routing, ExpressJS, I don't really have any strong opinion on this one. I guess one could also use hapi or something similar. Then for the frontend, for one, Angular2+ is my favourite and for another, it has the strongest Typescript community. The "N" in the stack is pretty obvious as there aren't any real viable alternatives for running JS on the server as far as I know. So overall the MEAN stack is surely one of the strongest choices if you want to go with typescript everywhere.

There's also a second dimension to this starterkit. While the first one was the lack of exemplary code for typescript on the server, the second one would be that every webapp is roughly the same. Every webapp has users. Every webapp has a login. Every webapp has a "main thing it revolves around" (posts for facebook, videos for youtube, images for instagram, todos for a todo list app, heroes in the hero app...). And the core of the webapp is usually providing "different lists of the main thing" (timeline, playlist, ...). Developing this core usually takes weeks of time. That's why there are things like Hackathon Starter Kits. But for one, there isn' any other good starter kit for a fullstack typescript webapp out there. And for another, this isn't something hacky. This is a starter for your next real webapp. It's got modularity, scalability and testability baked into it's design, so you can get started on the right foot.

Now go and check out that code and enjoy typescript day and night!
