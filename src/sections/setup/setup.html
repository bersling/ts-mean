Basic Setup

To get started, first create the folder structure. Create your project folder: 

mkdir project-name
cd project-name
mkdir backend
cd backend

Replace the red bits with your own values.

Then, first you'll need a package.json:

package.json 
{{> ./sections/setup/package.json.html}}

As we progress, we'll install some more packages, but that's it for now.

Now you're ready to install those dependencies. Simply run 

npm install

Now we're going to setup a logger and write a test for it. You could simply use console.log for logging, but with a logger you have some more functionalities at your disposal, such as disabling some logs under some circumstances (e.g. in production mode).

Create a the directory for the logger 

mkdir src
cd src
mkdir logger

and also the logger file and corresponding test:

src/logger/logger.ts
{{> ./sections/setup/logger.ts.html}}
src/logger/logger.test.ts
{{> ./sections/setup/logger.test.ts.html}}

Maybe some of you ask now "but why do I need this wrapper, I could use winston directly?". Well, in my experience it almost always paid of to write a wrapper. Like this, you can switch out the implementation of the logger easily. For example when winston becomes deprecated, switching to another logger library is much simpler. Also working with typescript and IntelliJ makes this quite comfortable since writing log and then hitting alt + enter will automatically import the logger!

Now that you've finished this part, let's check if everything is working. Run 

  npm test
