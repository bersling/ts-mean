App Config

Our first bit of source code will be a setter and getter for the application configuration. Go to the backend directory and create a folder named src and inside of this create a directory config: 

mkdir src
cd src
mkdir config

Then in this, folder, we're going to create the following three files:

src/config/app-properties.model.ts
{{> ./sections/app-config/app-properties.model.ts.html}}
src/config/AppConfig.ts
{{> ./sections/app-config/AppConfig.ts.html}}
src/config/appConfig.test.ts
{{> ./sections/app-config/appConfig.test.ts.html}}

Finally, as you can see, the config requires a properties file. Seprating the actual config makes the code more modular and easier to read. So create a folder properties inside of the src directory. There, create two files: A local.properties.json and a test.properties.json.

They look like that:

local.properties.ts
{{> ./sections/app-config/local.properties.json.html}}
test.properties.ts
{{> ./sections/app-config/test.properties.json.html}}

Note: I didn't leave the passwords in there by mistake. I did it so you could get started faster with a running MongoDB. However, once you get comfortable with a running setup, please go to mlabs and setup your own MongoDB.

Now that you've finished this part, let's check if everything is working. Run 

  npm test
